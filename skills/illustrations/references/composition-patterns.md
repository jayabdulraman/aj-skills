# Composition Patterns And Originality Rules

## Basic Structure Types

Choose one structure. Do not mix too many.

### Workflow

Good for: input -> processing -> output, content production, AI workflows, automation chains.

Drawing approach: input on the left, Xiaohei or a strange machine processing in the center, output on the right, with orange arrows showing the main flow.

### System Slice

Good for: information sources, filters, databases, renderers, agent system components.

Drawing approach: show only 3-5 core modules. Xiaohei participates in one key action.

### Before/After Contrast

Good for: messy/ordered, manual/automated, scattered/consolidated, anxious/stable.

Drawing approach: chaos on the left, stability on the right, orange arrow in the middle. Xiaohei can be a bit more exaggerated.

### Character State

Good for: user pain, creator state, too many tools, information anxiety, stuck-to-moving transitions.

Drawing approach: 2-4 small states, each with one short label.

### Concept Metaphor

Good for: content factory, information warehouse, mental black box, workflow machine, automated daily report.

Drawing approach: one large strange object or machine, a few inputs, one output. It should be memorable.

### Method Layers

Good for: methodology frameworks, system layers, capability stacks, content system layers.

Drawing approach: layered boxes rather than a formal pyramid. Xiaohei builds, carries, or assembles the layers.

### Map Route

Good for: idea-to-launch, user journey, content handoff path, learning route.

Drawing approach: one winding path with a few nodes. Xiaohei pulls a line or walks the path.

### Mini Comic

Good for: failure-to-success, a real process, dry commentary, before/after use.

Drawing approach: 2-4 small panels, each showing only one action.

## Original Metaphor Method

Invent a metaphor from the current writing every time. Do not copy old examples.

### Three Steps

1. Turn the abstract idea into a physical action: getting stuck, leaking, becoming heavy, sorting, settling, fermenting, opening, folding, unpacking, flowing back.
2. Turn the system structure into a low-tech object: broken machine, cardboard box, drawer, pipe, mailbox, odd dial, scale, well, ladder, strange workstation.
3. Make Xiaohei perform the action: stuck in the machine, pulling the wrong line, guarding a door, carrying, repairing, weighing, holding a ladder, recording, or stuffing something into a strange device.

### Object Pool

- Cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, pipe, tangled thread, gate, turntable, black box, hole punch, pasta press, clothesline, strange workstation.
- Pick only 1-2 at a time. Do not fill the canvas with props.

### Xiaohei Action Pool

- Pull, carry, stuff, scoop, press, weigh, sew, cut, twist, guard, push, catch, disassemble, mark, recycle.
- The action must serve the core meaning. Do not be strange for its own sake.

## Anti-Copy Rules

Do not open or copy `assets/examples/` by default. These images are only for style calibration: line density, whitespace, color restraint, and Xiaohei's role in the scene.

Unless the user explicitly asks to match or remake a specific example, do not reuse these old compositions:

- Conveyor with two breakpoints.
- Xiaohei pulling a judgment lever inside a content machine.
- Xiaohei becoming a funnel that sorts traffic, trust, and conversion.
- Xiaohei cutting a source-fish into many uses.
- Xiaohei pulling a handoff path.
- Xiaohei pulling three layers of information sources.
- Three Xiaohei figures each doing traffic/trust/conversion jobs.
- Xiaohei stamping a handoff-copy toolbox.
- Xiaohei holding a warning sign near a pitfall path.

Even for similar topics, invent a new metaphor. A handoff path does not have to be a road; Xiaohei could attach a content tail to a doorknob. Reuse from one source does not have to use a fish; Xiaohei could press one paper ball into several shapes.
