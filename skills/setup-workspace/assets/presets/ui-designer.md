# Preset: UI Designer

## When to use
Role is UI/visual/product designer. Produces mockups, prototypes, design specs, design-system components; works in Figma; hands off to engineering.

## Suggested structure
```
projects/                 # one folder per product/feature
  <project>/
    explorations/  mockups/  prototypes/  specs/ (handoff)  assets/  feedback/
design-system/            # components, tokens, patterns
templates/  work-memory/  connectors/  resources/  archive/
```

## Recommended templates
design-brief · design-spec (handoff) · component-doc · critique-notes · accessibility-review

## AGENTS.md role snippet
> Focus areas: visual/interaction design, prototyping, design-system consistency, handoff. Delegate: drafting specs, documenting components, critique summaries, accessibility checks. Apply the org's brand/design system by default; flag off-brand or non-AA work.

## Tailoring notes
Ask their design tool (Figma, etc.), handoff target, and brand/design system. If the org has a design-system skill (e.g. `edplus-asu-design-system`), wire it in so specs and prototypes come out on-brand and accessible.
